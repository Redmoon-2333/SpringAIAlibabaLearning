Kubernetes 是一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用程序。
Prometheus 是一个开源的监控和告警系统，专门用于记录时间序列数据。它具有多维数据模型和强大的查询语言 PromQL。
Grafana 是一个开源的数据可视化和监控平台，可以与 Prometheus、Elasticsearch 等数据源集成。
Docker 是一个开源的容器化平台，允许开发者将应用及其依赖打包到轻量级、可移植的容器中。
微服务架构是一种将应用程序构建为一组小型、独立服务的方法，每个服务运行在自己的进程中。
日志分析是 AiOps 的重要组成部分，通过分析系统日志可以快速定位问题和预测潜在故障。
CI/CD 是持续集成和持续部署的缩写，是现代 DevOps 实践的核心，可以自动化软件交付流程。
服务网格 Service Mesh 如 Istio 提供了微服务间通信的基础设施层，包括流量管理、安全和可观测性。
Elasticsearch 是一个分布式搜索和分析引擎，常用于日志分析和全文搜索场景。
告警疲劳是指运维人员因为接收过多告警而导致的敏感度下降，需要通过智能告警聚合和去重来解决。
APM（Application Performance Monitoring）应用性能监控用于实时监测应用程序的性能指标，帮助快速发现和诊断性能问题。
分布式追踪系统如 Jaeger 和 Zipkin 可以追踪请求在微服务架构中的完整调用链路，帮助定位性能瓶颈。
日志聚合工具如 ELK Stack（Elasticsearch、Logstash、Kibana）可以集中收集、存储和分析来自多个源的日志数据。
容器编排中的 Pod 是 Kubernetes 中最小的部署单元，可以包含一个或多个紧密协作的容器。
自动扩缩容 HPA（Horizontal Pod Autoscaler）可以根据 CPU 使用率或其他指标自动调整 Pod 的数量。
蓝绿部署是一种发布策略，通过维护两个相同的生产环境来实现零停机部署和快速回滚。
金丝雀发布是一种渐进式发布策略，先将新版本部署到小部分用户，验证无误后再全量发布。
Helm 是 Kubernetes 的包管理工具，可以简化应用程序的部署和管理，提供版本控制和回滚功能。
配置中心如 Consul 和 Nacos 提供了集中式的配置管理，支持配置动态更新和多环境管理。
服务发现机制允许微服务自动发现和调用其他服务，常见实现包括 Consul、Eureka 和 Kubernetes Service。
熔断器模式如 Hystrix 可以防止故障在微服务系统中级联传播，提高系统的容错能力。
限流和降级是保护系统的重要手段，可以在高负载情况下保证核心服务的可用性。
SRE（Site Reliability Engineering）站点可靠性工程强调通过软件工程方法来解决运维问题。
SLO（Service Level Objective）服务级别目标定义了服务应该达到的可靠性目标，是 SRE 实践的核心。
错误预算是 SLO 允许的最大故障时间，用于平衡系统可靠性和功能开发速度。
混沌工程是一种主动测试系统韧性的方法，通过在生产环境中引入故障来验证系统的容错能力。
可观测性三支柱包括指标（Metrics）、日志（Logs）和追踪（Traces），是现代运维监控的基础。
基础设施即代码 IaC（Infrastructure as Code）使用代码来定义和管理基础设施，提高了部署的可重复性和一致性。
Terraform 是一个流行的 IaC 工具，支持多云环境下的基础设施管理和自动化部署。
容器镜像扫描可以检测镜像中的安全漏洞和恶意代码，是容器安全的重要环节。
